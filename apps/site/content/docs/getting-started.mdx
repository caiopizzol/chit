---

chit runs under Bun. The npm package ships the CLI; the source repo ships example manifests and Studio client assets.

## Install

```sh
bunx @chit-run/cli --help
```

Or install the `chit` binary:

```sh
bun install -g @chit-run/cli
chit --help
```

## Write or inspect a chit

A chit is a JSON manifest. Point the CLI at one:

```sh
chit show path/to/your-manifest.json
chit run path/to/your-manifest.json --input question="..."
```

## Run the repo example

The repo ships a small first-run manifest in `examples/consult.json`. It fans a question out to Codex and Claude, then formats both answers.

```sh
git clone https://github.com/caiopizzol/chit
cd chit
bun install
bun run cli run examples/consult.json --scope work-session
```

The run prints what ran: the resolved manifest, the adapters, the validation result, and the execution levels. No chat log.

## Read a chit before you run it

`show` renders a manifest without executing it, in any of four views.

```sh
bun run cli show examples/consult.json            # ASCII graph
bun run cli show examples/consult.json --json     # JSON
bun run cli show examples/consult.json --mermaid  # Mermaid
```

## Open Studio

Studio is a local web editor for chits: a graph view, an inspector, and a read-only Loops drawer.

```sh
bun run cli studio examples/consult.json
```

Studio currently needs a source checkout; the published CLI package does not include the Studio client bundle.
