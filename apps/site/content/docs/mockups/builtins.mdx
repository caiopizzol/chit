## Profiles

	<Tab>`{ "critic": "gemini" }` uses the adapter's default model.</Tab>

	<Tab>`{ "judge": { "adapter": "claude", "model": "haiku" } }` writes the binding out. Custom adapters must use this form.</Tab>

## Steps

<Tabs items={["call", "ask", "check", "format", "routine"]}>
	<Tab>`id`, `call`, `prompt`. Ask a routine agent; stores its text output.</Tab>
	<Tab>`id`, `ask`. Pause for one operator answer, fed forward, never stored.</Tab>
	<Tab>`id`, `check`. Run one or more commands. Any check forces a sandbox.</Tab>
	<Tab>`id`, `format`. Assemble text from templates and prior output.</Tab>
	<Tab>`id`, `routine`, `inputs`. Run another routine and store its output.</Tab>
</Tabs>

## Loops

<Tabs items={["checks-pass", "step equals", "all"]}>
	<Tab>`"checks-pass"` is deterministic and the default. May omit the cap; it defaults to 5.</Tab>
	<Tab>`{ "step": "review", "equals": "pass" }` lets a verdict end the loop. The cap is required.</Tab>
	<Tab>`{ "all": ["checks-pass", { "step": "review", "equals": "pass" }] }` needs both. A reviewer can block convergence.</Tab>
</Tabs>

## What the parser rejects

<Accordions type="single">
	<Accordion title="codex:sonnet">sonnet is a Claude model, not a codex one.</Accordion>
	<Accordion title="codex: (trailing colon)">No model after the colon. Use `codex` for the default.</Accordion>
	<Accordion title="ollama:llama3">Unknown adapter. Custom adapters must use the object form.</Accordion>
	<Accordion title="a judged loop with no maxIterations">A judged condition has no guaranteed termination, so the author must cap it.</Accordion>
</Accordions>
