## What to look for

The current config page is mostly markdown tables and code blocks stacked vertically. The question is which fields want a typed reference (a table), which want alternatives side by side (tabs or cards), and which want a picture (the derived behavior, the filesystem escalation, the annotated example).

## Recommendation

**Hybrid.** `TypeTable` is the right tool for field references: typed, scannable, framework-native, and it themes to the paper-and-ink palette for free. A few custom visuals carry what a table cannot say well: the "how it runs is derived" rule, the filesystem escalation, and the annotated example. Pure built-ins read a little generic; pure custom is more to maintain than it returns.
